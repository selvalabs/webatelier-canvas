# Definição de pronto

Uma tarefa só está pronta quando:

- issue criada;
- branch dedicada criada a partir da main atualizada;
- implementação dentro do escopo;
- testes adicionados ou atualizados;
- documentação atualizada quando necessário;
- lint passou;
- testes passaram;
- typecheck Python passou;
- typecheck/build TypeScript passaram;
- diff revisado;
- sem segredos ou dados reais;
- commit segue Conventional Commits;
- PR referencia a issue;
- CI está verde;
- revisão humana foi concluída;
- merge foi confirmado pelo responsável;
- main foi sincronizada localmente;
- branch foi removida;
- validação final passou.
